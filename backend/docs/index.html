GymSaaS
Backend API

API REST para la gestión de gimnasios como servicio. Maneja autenticación de usuarios, registro de gyms, perfiles de clientes y autorización mediante JWT.

ℹ️ Base URL de desarrollo: http://localhost:3000

Stack tecnológico

RuntimeNode.js + ESM
FrameworkExpress 5
Base de datosPostgreSQL (Supabase)
ORM / Querypostgres.js
AuthJWT + bcrypt
Dev toolnodemon

Configurar en una nueva máquina

Clonar el repositorio

git clone <repo-url>

Instalar dependencias

npm run build:backend
npm run build:frontend

Crear el archivo .env

Crea un archivo .env en la raíz de /backend con estas variables:

# Conexión a PostgreSQL (Supabase Pooler recomendado)
DATABASE_URL=postgresql://usuario:password@host:puerto/postgres

# Puerto del servidor
PORT=3000

# Secreto para firmar JWT (usa un UUID o string seguro)
JWT_SECRET=tu-secreto-superseguro
⚠️ El archivo conecta directamente a la base de datos de producción. Nunca compartas tu .env real ni lo subas a Git. Agrégalo a .gitignore.

Crear las tablas en PostgreSQL

Ejecuta este SQL en tu base de datos (Supabase SQL Editor o psql):

CREATE TABLE gyms (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name       TEXT UNIQUE NOT NULL,
  address    TEXT,
  phone      TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE users (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  gym_id     UUID REFERENCES gyms(id) ON DELETE CASCADE,
  name       TEXT NOT NULL,
  email      TEXT UNIQUE NOT NULL,
  password   TEXT NOT NULL,
  role       TEXT DEFAULT 'client',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE clients_profile (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  gym_id            UUID REFERENCES gyms(id) ON DELETE CASCADE,
  user_id           UUID REFERENCES users(id) ON DELETE CASCADE,
  phone             TEXT,
  address           TEXT,
  eps               TEXT,
  emergency_contact TEXT,
  created_at        TIMESTAMPTZ DEFAULT NOW()
);

Ejecutar el servidor

# Desde la raíz del proyecto
npm run start:backend

# Salida esperada:
Servidor corriendo en puerto 3000
DB conectada: [ { now: 2026-... } ]
✅ Si ves ambos mensajes, el servidor está listo y la DB conectada correctamente.

Auth — /api/auth

Endpoints para autenticación de usuarios y gestión de cuentas.

POST Público /api/auth/register

Registra un nuevo usuario. Si el rol es client, también crea su perfil en clients_profile.

Body (JSON)

CampoTipoRequeridoDescripción
namestring✅Nombre del usuario
emailstring✅Email único
passwordstring✅Se hashea con bcrypt
gym_idUUID✅ID del gym al que pertenece
rolestring—client (default) o admin
phonestring—Solo para role=client
addressstring—Solo para role=client
epsstring—Solo para role=client
emergency_contactstring—Solo para role=client

Clients — /api/clients

Todos los endpoints de clientes requieren JWT.

GET 🔒 JWT /api/clients/:gym_id

Devuelve todos los clientes de un gym, con su perfil y datos de usuario.

Respuesta — 200

[
  {
    "id": "uuid...",
    "user_id": "uuid...",
    "phone": "300...",

    "eps": "Sura",

  }
]
GET 🔒 JWT /api/clients/client/:id_client

Devuelve un cliente por su ID de perfil (no el user_id).

PUT 🔒 JWT /api/clients/client/:id_client

Actualiza el perfil de un cliente.

CampoTipo
phonestring
addressstring
epsstring
emergency_contactstring
DELETE 🔒 JWT /api/clients/client/:id_user

Elimina el usuario (y su perfil en cascada). El parámetro es el user_id, no el id_client.

Autenticación con JWT

Los endpoints protegidos requieren el token en el header Authorization:

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

El middleware decodifica el token e inyecta req.user con { id, role } para uso en los controladores.

CódigoMotivo
401No se envió el token
403Token inválido o expirado

Uso desde el frontend

Guía práctica para consumir la API desde cualquier frontend (Vue, React, etc.).

1. Login y guardar el token

// authService.js
const API = 'http://localhost:3000/api'

export async function login(email, password) {
  const res = await fetch(`${API}/auth/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email, password })
  })

  if (!res.ok) throw new Error('Credenciales inválidas')

  const data = await res.json()
  localStorage.setItem('token', data.token)
  localStorage.setItem('user', JSON.stringify(data.user))
  return data
}

2. Helper para peticiones autenticadas

// api.js
const API = 'http://localhost:3000/api'

function authHeaders() {
  const token = localStorage.getItem('token')
  return {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  }
}

export async function apiFetch(endpoint, options = {}) {
  const res = await fetch(`${API}${endpoint}`, {
    ...options,
    headers: { ...authHeaders(), ...options.headers }
  })
  if (!res.ok) {
    const err = await res.json()
    throw new Error(err.message || 'Error en la petición')
  }
  return res.json()
}

Flujo completo de login en React

// LoginPage.jsx
import { useState } from 'react'
import { login } from './authService.js'

export default function LoginPage() {
  const [email, setEmail] = useState('')
  const [pass, setPass]  = useState('')
  const [error, setError] = useState(null)

  async function handleSubmit(e) {
    e.preventDefault()
    try {
      await login(email, pass)
      window.location.href = '/dashboard'
    } catch (err) {
      setError(err.message)
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={email} onChange={e => setEmail(e.target.value)} />
      <input type="password" value={pass} onChange={e => setPass(e.target.value)} />
      {error && <p style={{color:'red'}}>{error}</p>}
      <button type="submit">Entrar</button>
    </form>
  )
}

404Not FoundUsuario, cliente o gym no existe

⚠️ Todos los errores devuelven JSON con un campo message o error descriptivo.
